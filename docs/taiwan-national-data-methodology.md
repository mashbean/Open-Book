# 全台版資料方法

## 共同基準

全台首頁採用行政院主計總處發布的 [115 年度直轄市及縣市總預算彙編](https://www.dgbas.gov.tw/News_Content.aspx?n=1525&s=236271)。這份工作簿提供 22 縣市一致口徑的歲入、歲出、融資、債務還本、經常支出、資本支出及八類政事別資料，適合做跨縣市總覽。

所有金額沿用來源工作簿的千元新臺幣。英文版依網站既有規則，以 `NT$32.25 = US$1` 顯示美元估算。金門縣在彙編中是總預算案，其他縣市依主計總處表註呈現。

## 自動更新與驗證

執行以下指令會下載官方 Excel 並重建全台資料。

```bash
npm run refresh:national
```

`scripts/build-national-data.mjs` 會優先使用系統 `curl` 的信任鏈下載官方附件，環境沒有 `curl` 時才改用 Node `fetch`。下載程序不會關閉 TLS 憑證驗證。完成下載後會執行四項資料驗證。

- 必須取得 22 個縣市的歲入與歲出總額
- 每個縣市的八類政事別加總必須等於歲出總額
- 經常支出與資本支出加總必須等於歲出總額
- 歲入、舉借與以前年度賸餘，扣除歲出及債務償還後，必須等於來源表的預計收支賸餘

產出檔為 `pages-site/cities.json`。原始 Excel 不納入版本控制，來源網址、下載網址、產生時間、單位與資料註記都保留在 JSON 的 `meta` 欄位。

## 地方資料成熟度

共同彙編可以回答總體配置，無法直接支撐逐機關、逐計畫或逐筆支付查詢。網站另用四個面向評估從地方官方來源持續維護深度頁的難度。

| 面向 | 評估方式 |
| --- | --- |
| 官方機器格式 | 完整歲入與歲出 4 分，部分表格 2 分，只有文件下載 0 分 |
| 歷年資料 | 8 年以上 2 分，4 至 7 年 1 分，3 年以下 0 分 |
| 單位預算明細 | 官方完整公布 2 分，民代協助取得 1 分，尚未取得 0 分 |
| 穩定介面 | 公開 API 2 分，穩定機器檔 1 分，文件頁 0 分 |

總分 7 分以上列為 A 級，可優先製作自動更新的城市深度頁。1 至 6 分列為 B 級，適合先做共同總覽，再補欄位對照或文件解析。0 分列為 C 級，需要先改善官方來源或建立人工整理流程。

評分資料保存在 `data-sources/city-readiness.json`，包含每個縣市的官方入口、OpenFun 歷年覆蓋、單位預算明細狀態與介面類型。評分日期為 2026 年 7 月 22 日，來源異動後應重新查核。

## OpenFun 與 Twinkle 的 22 縣市實測

`scripts/audit-ai-interfaces.mjs` 會執行兩種不同驗收。

```bash
set -a
source ~/.config/secrets/twinkle.env
set +a
npm run refresh:interfaces
```

OpenFun 使用免 token 的 `tw.openfun~bulk~budget-detail` 靜態檔案。程式逐縣市下載 `工作計劃.csv`，統計年度、列數、最新年度最大工作計畫與最大年度變動，並用 HEAD 檢查 `歲出分支列表.json` 是否真的有計畫文字與用途別內容。Twinkle 使用 Bearer token 連接正式 MCP，對每個縣市各執行一次廣泛預算搜尋與一次全縣市歲出搜尋，再讀取最佳命中的 dataset metadata。若最佳命中為全縣市歲出候選，管線會進一步呼叫 `query_rows` 取得實際資料列，依明確欄位規則計算總額，最後和主計總處 115 年共同表對帳。

2026 年 7 月 23 日的實測結果如下。

- OpenFun 工作計畫 CSV 可讀取 21 個縣市，彰化縣的通用 URL 回傳 404
- OpenFun 的詳細歲出 JSON 在 21 個縣市回傳成功，但只有 11 個檔案有實際內容，其餘多為 2-byte 空物件
- 澎湖縣工作計畫最新只到民國 110 年，其他可讀縣市多已到 115 年
- Twinkle 的 22 縣市 MCP 呼叫全部成功，當次查詢成本皆顯示 `US$0`
- 21 個縣市的最佳命中屬於同一縣市，只有 8 個最佳命中確實是全縣市歲出總預算
- 8 個候選皆成功呼叫 `query_rows` 並取得實際資料列；只有新北市與嘉義市的回傳總額和官方 115 年歲出完全相符
- 新北市使用 63 列資料並以標題確認 115 年度；嘉義市使用 13 列資料，總額完全相符，但來源未明示年度，因此網站會保留此限制
- 桃園、臺南、高雄與南投的回傳總額分別和官方基準相差約 46.7%、20.8%、14.3% 與 4.1%，未納入正式明細
- 臺中資料列日期為 2019 年，花蓮金額欄明示民國 107 年，兩者均判定為舊資料
- Twinkle 文件列出的工具名稱帶 `opendata-` 前綴，正式工具 registry 實際回傳無前綴名稱
- 新竹縣最佳命中的 CSV schema 出現編碼亂碼，臺中全市歲出資料集 metadata 只有 2 列，顯示命中標題後仍需驗證內容

結果保存在 `data-sources/ai-interface-audit.json`，包含查詢摘要、metadata、讀列結果與衍生統計，不含 Twinkle API key。`npm run refresh:national` 會執行官方總額對帳並合併進 `pages-site/cities.json`。GitHub Pages 只讀取已驗證快照，不需要接觸 API key。

這套檢查刻意分開六個層次，包括服務可連線、搜尋有結果、命中同一縣市、命中全縣市候選、實際資料列可讀，以及總額和官方共同表完全相符。只有最後一層可以供應機關別補充明細，官方總額本身不會被 TwinkleAI 取代。

## 值得追問的訊號如何產生

網站目前使用兩組可重現訊號。官方共同表提供舉借、債務償還、以前年度賸餘、資本支出與政事別占比；OpenFun 工作計畫提供最新年度與前一年度的科目變動。訊號只負責指出值得回原始預算書查證的位置。

例如臺中市 115 年編列舉借 1,143.38165 億元、債務償還 1,000 億元，兩者毛額合計超過歲出規模；新竹市的工商管理工作計畫由 7,405 萬元增至 25.08533 億元；臺南市衛生業務增加約 1,039%，同時最大社福工作計畫減少約 39%。這些數字可能來自換債、基金移轉、中央補助、組織調整、科目重分類或一次性工程，不能單靠幅度判定浪費或不當。

## 適合下一階段製作的縣市

臺北以外，新北、桃園、臺中、臺南與高雄已有較完整的機器格式或資料介面，適合建立與臺北相近的深度頁。OpenFun 實測顯示臺南、新竹市、南投、雲林、嘉義縣、屏東、花蓮、臺東、金門與連江等地有非空的詳細歲出 JSON，可發展逐計畫文字搜尋。新北、桃園、臺中與高雄雖有可用工作計畫 CSV，詳細 JSON 目前是空物件，需先確認來源與更新流程。

## 重要界線

成熟度分數衡量的是維護 OpenBook 深度頁的工程條件，不代表地方政府是否依法公開預算，也不代表公民科技社群是否已整理出更細資料。

預算數字描述政府獲准如何配置資源。採購契約描述政府向誰採購及契約上限，決算與支付資料才更接近實際執行。三者不能混作同一欄位或以契約金額推論實際付款。
