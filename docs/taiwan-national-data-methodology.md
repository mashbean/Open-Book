# 全台版資料方法

## 共同基準

全台首頁採用行政院主計總處發布的 [115 年度直轄市及縣市總預算彙編](https://www.dgbas.gov.tw/News_Content.aspx?n=1525&s=236271)。這份工作簿提供 22 縣市一致口徑的歲入、歲出、融資、債務還本、經常支出、資本支出及八類政事別資料，適合做跨縣市總覽。

所有金額沿用來源工作簿的千元新臺幣。英文版依網站既有規則，以 `NT$32.25 = US$1` 顯示美元估算。金門縣在彙編中是總預算案，其他縣市依主計總處表註呈現。

## 自動更新與驗證

執行以下指令會下載官方 Excel 並重建全台資料。

```bash
npm run refresh:national
```

`scripts/build-national-data.mjs` 會優先使用系統 `curl` 的信任鏈下載官方附件，環境沒有 `curl` 時才改用 Node `fetch`。下載程序不會關閉 TLS 憑證驗證。完成下載後會執行三項資料驗證。

- 必須取得 22 個縣市的歲入與歲出總額
- 每個縣市的八類政事別加總必須等於歲出總額
- 經常支出與資本支出加總必須等於歲出總額

產出檔為 `pages-site/cities.json`。原始 Excel 不納入版本控制，來源網址、下載網址、產生時間、單位與資料註記都保留在 JSON 的 `meta` 欄位。

## 地方資料成熟度

共同彙編可以回答總體配置，無法直接支撐逐機關、逐計畫或逐筆支付查詢。網站另用四個面向評估從地方官方來源持續維護深度頁的難度。

| 面向 | 評估方式 |
| --- | --- |
| 官方機器格式 | 完整歲入與歲出 4 分，部分表格 2 分，只有文件下載 0 分 |
| 歷年資料 | 8 年以上 2 分，4 至 7 年 1 分，3 年以下 0 分 |
| 單位預算明細 | 官方完整公布 2 分，民代協助取得 1 分，尚未取得 0 分 |
| 穩定介面 | 公開 API 2 分，穩定機器檔 1 分，文件頁 0 分 |

總分 7 分以上列為 A 級，可優先製作自動更新的城市深度頁。1 至 6 分列為 B 級，適合先做共同總覽，再補欄位對照或文件解析。0 分列為 C 級，需要先改善官方來源或建立人工整理流程。

評分資料保存在 `data-sources/city-readiness.json`，包含每個縣市的官方入口、OpenFun 歷年覆蓋、單位預算明細狀態與介面類型。評分日期為 2026 年 7 月 22 日，來源異動後應重新查核。

## 適合下一階段製作的縣市

臺北以外，新北、桃園、臺中、臺南與高雄已有較完整的機器格式或資料介面，適合建立與臺北相近的深度頁。雲林、金門與連江雖然自動化程度較低，但 OpenFun 的盤點顯示官方有較完整的單位預算明細，適合發展逐機關與逐計畫查詢。新竹市與臺南另有民代協助取得明細的案例，可作為資料合作路線。

## 重要界線

成熟度分數衡量的是維護 OpenBook 深度頁的工程條件，不代表地方政府是否依法公開預算，也不代表公民科技社群是否已整理出更細資料。

預算數字描述政府獲准如何配置資源。採購契約描述政府向誰採購及契約上限，決算與支付資料才更接近實際執行。三者不能混作同一欄位或以契約金額推論實際付款。
