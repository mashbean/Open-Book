# 臺灣版 OpenBook 資料範圍

## 第一版選擇地方政府

OpenBook 的核心是讓居民理解一個行政區的預算，而不是做全國財政統計入口。地方政府的服務範圍較貼近日常生活，主管機關數量也足以放在單一入口中探索。中央政府的資料規模更大，但目前公開內容多是跨機關彙總、各機關各自發布的預決算書表，第一版若直接納入中央層級，會先遇到資料標準化問題。

臺北市適合作為第一個實作對象，因為主計處已持續發布機器可讀的歲入與歲出 XML，欄位結構跨年度一致，也包含經常門、資本門與機關別分類。

## 使用的官方資料

- [臺北市總預算歲出機關別預算總表](https://data.gov.tw/dataset/132907)，採用 114、115 年度正式總預算 XML
- [臺北市總預算歲入來源別預算總表](https://data.gov.tw/dataset/132917)，採用 114、115 年度正式總預算 XML
- [臺北市總預決算歲出按政事別](https://data.gov.tw/dataset/138490)，作為後續加入預算與決算長期比較的資料源
- [臺北市政府歲入預算執行情形](https://data.gov.tw/dataset/121177)，作為後續加入年度執行率的資料源

資料採用政府資料開放授權條款第 1 版。`scripts/refresh-taipei-data.mjs` 會直接從臺北市資料大平臺下載來源 XML，重新產生 `sample-data/taipei-*.csv`。

## 現階段能回答的問題

- 115 年度總歲出有多少
- 經常門與資本門各占多少
- 哪些主管機關的預算最高
- 115 年度總歲入從哪些來源而來
- 各分類相較 114 年度增加或減少多少
- 各主管機關的資本門預算規模

## 現階段不能回答的問題

目前官方資料是預算彙總，並非逐筆付款帳。網站無法據此回答某日付給哪一家廠商、發票內容或付款憑證。政府採購決標資料可以補上得標廠商與契約金額，但契約金額不等於實際支付，不應混為同一欄位。

下一階段若要接近美國 open checkbook 的查帳深度，需要地方政府另外開放去識別化後的支付明細，至少包含付款日期、支用機關、會計科目、受款對象、金額、契約或採購案號。
